Project idea

Overview

A key logger logs all keystrokes made on your computer. The information collected by a keylogger - which can include key stokes, clipboards, screenshots - can be used to steal information such as passwords and other personal information. Often logged information can be transmitted over the internet.

I propose a desktop application ("the application") for personal use that creates a secure container in which sensitive websites are accessed. The target audience is people who are less computer savvy, and who are more likely to download malware and less likely to detect its presence on the computer.

Motivation

In a post on the InfoSec website the author of a post entitled Keyloggers: How they work and more3, refers to statistics showing that in 2005 as much as 15% of corporate computes may be infected with a keylogger. I was not able to find statistics on the rate of infection for personal computers; however, it is easy to imagine that most personal computers are less secure than those managed by corporations.

According to the Australian Bureau of Statistics website an estimated 126, 300 people experienced identity fraud in 2014-154. In the same report states that 1.1 million people experienced card fraud, of which 267,800 people were aged 55 and over. Total losses were estimated at around $2.1 billion dollars.

As we move more online, more personal information will be provided by people to various public and private institutions. This means more opportunities for hackers to use tools such as keyloggers to steal personal information.

Product description

As noted above the application creates a secure container within which sensitive websites can safely be accessed. The contents of the container can be displayed on a screen. However, clipboard and screenshot functionality would be unavailable inside the container. Key stroke information could not be passed from within to outside the container

Installation would be handled by a standard installer package.

Depending on how locked down the application needs to be it is possible that the user could see the application as quite restrictive. On the other hand, the application is designed to do one thing - stop information within the container being passed logged by a keylogger.

It will be important to limit the functionality of the product, in the sense of what it can be used for. This will help keep the application and information entered by the user secure. This will, with the help of experts in user-centred design, also help to develop a product that is as simple to use as possible. The less it does the less processes, menus and buttons are required to be catered for.

Tools and Technologies

This project will require application project management tools, such as JIRA, and software development tools like Eclipse.

Skills Required

The project will require people with skills in the relevant programming language to build the software. Also required will be people with an in-depth knowledge of computer hardware and operating systems such as Windows. People with skills in user-centred design will help to ensure the application is as simple to use as possible. Finally, project management staff (project manager, finance etc.) will be required to manage the project.

Outcome

If successful this project will deliver a simple to use desktop application that people can use to input sensitive personal information or passwords safely into websites. The application will act as an additional safeguard against identity and password theft.
